MessAdmin

Notification system and Session administration for J2EE Web Applications

Introduction

MessAdmin is a utility for viewing and interacting with Java HttpSession. MessAdmin can be added to any J2EE application, with zero change to the monitored application!
With MessAdmin, you can:
  • display statistics about a Web Application
  • monitor every active HttpSession in a web application
  • see detailed information about any session
  • remove any attribute from any session
  • manually expire any session
  • send a html message to any user of the application, or all of them
MessAdmin can be added to any J2EE application. And apart from a very minor change to your web.xml, you don't need to change anything at all in your web application! That's right, no coding, no application change!
Administration is done either with a built-in servlet, or via JMX (if available).

Full feature list

See and interact with MessAdmin via
  • built-in administration servlet
  • JMX (if available)
WebApp-related statistics and operations:
  • number of active Sessions
  • number of passive Sessions
  • maximum number of concurrent Sessions (peak), with date at which this event happened
  • total number of created Sessions
  • total number of hits
  • total Request size (incoming bandwidth)
  • total Response size (outgoing bandwidth)
  • list of ServletContext (application-scope) attributes, with size and remove operation
Session-related statistics and operations:
  • remote user name
  • remote user locale
  • remote host IP
  • last Request URL
  • isSecure flag (e.g. https)
  • date created
  • date of last access
  • TTL (time to live before expiration)
  • total number of hits
  • total Request size
  • total Response size
  • last Request size and date
  • last Response size and date
  • list of HttpSession attributes, with size and remove operation
  • send an HTML message to all or any user of the application
  • invalidate any HttpSession
As a bonus, MessAdmin takes care of some evil singleton cleanup at application shutdown, thereby avoiding some dreadful OOM (java.lang.OutOfMemoryError) on hot restart:
  • JDBC drivers: java.sql.DriverManager - Evil Singleton (drivers should be located at application-server level)
  • java.beans.Introspector - Slightly less Evil singleton
  • Jakarta Commons Logging <= 1.0.4
You will still have to take care of your own ThreadLocals; there's nothing I can do for you there...

Requirements

MessAdmin needs at least Java 1.3, and a Java Servlet/JSP container (Servlet 2.3/J2EE 1.3). JMX administration requires Java 5.

Installation

Please refer to the documentation (INSTALL.txt) in the package for detailed instructions. You can find the download at the SourceForge project page.
You will have to:
  • copy MessAdmin.jar in your WEB-INF/lib/ directory
  • copy the MessAdmin folder (containing the administration pages) into your document root directory
  • copy some configuration to your web.xml
That's all! You do not need to modify the source code of your application. This is powerful stuff!
Some of the installation options can be customized. Please see the doc.

How stable is this product? How much memory/CPU does it use?

MessAdmin is already used in production sites. Its overhead is very small (less than 600 bytes per session) and doesn't make any heavy computation. With its plug'n'play capabilities, there is no reason to not add it to all your applications.

What's in stock for the future?

While MessAdmin has now reached a reasonably stable phase, we do have a stock of features we will eventually add over the time. Refer to the TODO.txt file in the package for an up-to-date list. Should you want to contribute either code or ideas, please do so!
